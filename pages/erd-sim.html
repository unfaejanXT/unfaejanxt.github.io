1 : 1
💡 Kuncinya di UNIQUE pada foreign key. Tanpa UNIQUE, relasi otomatis jadi 1:N.

Simulator

Tambah pegawai dulu, baru kasih kartu. Coba langgar aturannya!

SQL:
CREATE TABLE kartu_akses (
  id INT PRIMARY KEY,
  nomor VARCHAR(20),
  pegawai_id INT UNIQUE, -- ⭐ kunci 1:1
  FOREIGN KEY (pegawai_id) 
    REFERENCES pegawai(id)
);

Kesimpulan Pengaruh ke Tabel

Pilih relasi = tentukan struktur fisik database-mu.

Relasi Struktur Tabel Foreign Key Constraint Kunci Kapan Dipakai
One-to-One (1:1) 2 tabel FK + UNIQUE di salah satu UNIQUE Profil terpisah, data sensitif, opsi perluas (user ↔ profile)
One-to-Many (1:N) 2 tabel FK di sisi "banyak" INDEX biasa Paling umum! (pelanggan → pesanan, kategori → produk)
Many-to-Many (M:N) 3 tabel (1 junction) 2 FK di tabel tengah COMPOSITE PK Tagging, enrollment, likes, followers (siswa ↔ kelas)
Ingat santai: Gambar Crow's Foot dulu sebelum bikin tabel. Kalau ada "banyak" di kedua sisi → pasti butuh tabel ketiga. Kalau ada "satu" di kedua sisi → kasih UNIQUE.

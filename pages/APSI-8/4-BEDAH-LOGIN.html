Evolusi Kode: Membedah Form Login

Dalam Analisis dan Perancangan Sistem Informasi (APSI), antarmuka adalah jembatan antara user dan sistem. Kita akan membedah bagaimana elemen mentah HTML berubah menjadi UI profesional.

2. Bedah Struktur HTML (The Skeleton)

Berikut adalah kode HTML murni untuk form login tanpa gaya sedikitpun. Pahami peran setiap barisnya.

<!-- Pembungkus Utama -->
<div class="login-box">
    <h2>Login Sistem</h2>
    
    <form>
        <!-- Grup Input Username -->
        <div class="form-group">
            <label for="user">Username</label>
            <input type="text" id="user" required>
        </div>

        <!-- Grup Input Password -->
        <div class="form-group">
            <label for="pass">Password</label>
            <input type="password" id="pass" required>
        </div>

        <!-- Checkbox Ingat Saya -->
        <div class="form-check">
            <input type="checkbox" id="remember">
            <label for="remember">Ingat Saya</label>
        </div>

        <button type="submit">Masuk</button>
    </form>
</div>
ElemenFungsi Analitis
<form>Elemen penampung utama untuk mengumpulkan input data yang akan dikirim ke server.
<div class="login-box">Kontainer entitas form agar terpisah dari elemen latar belakang atau konten lain.
<label for="...">Menghubungkan teks deskriptif secara semantik dengan kolom input (Meningkatkan Aksesibilitas).
<input type="text">Menyediakan kolom isian baris tunggal untuk teks standar (contoh: Username).
<input type="password">Menjaga kerahasiaan dengan mengubah karakter ketikan menjadi titik/bintang (Security).
<input type="checkbox">Menyediakan pilihan biner (ya/tidak) untuk menyimpan sesi login pengguna (Ingat Saya).
<button type="submit">Tombol pemicu yang menjalankan aksi pengiriman data (submit) dari form ke server.
required (Atribut)Atribut validasi sisi klien yang memastikan kolom input tidak boleh dibiarkan kosong.

3. Tampilan Sementara (HTML Saja)

Tanpa CSS, browser akan merender elemen secara berurutan ke bawah (flow) dengan gaya default yang sangat sederhana.

*Inilah tampilan asli web sebelum diberikan 'sentuhan visual'.

4. Step-by-Step Penambahan CSS

Step 1 Reset & Centering (Flexbox)

Tujuannya membuang margin bawaan browser dan meletakkan form tepat di tengah layar. Flexbox sangat efektif untuk layouting elemen.

Properti CSSDeskripsi Fungsi
box-sizing: border-box;Memastikan penambahan padding dan border tidak mengubah total dimensi lebar/tinggi elemen.
display: flex;Mengubah <body> menjadi flex container untuk mempermudah pengaturan tata letak di dalamnya.
justify-content & align-itemsSecara bersamaan memposisikan konten agar berada tepat di tengah secara horizontal dan vertikal.
height: 100vh;Mengatur tinggi elemen minimal sama dengan 100% tinggi layar perangkat (Viewport Height).
 
* { box-sizing: border-box; margin: 0; padding: 0; }
body {
    display: flex; justify-content: center; align-items: center;
    height: 100vh; background-color: #f2f2f2; font-family: sans-serif;
}
Step 2 Membuat Kartu (The Container)

Memberikan batas visual, warna putih, dan bayangan lembut agar form terlihat menonjol dan terpisah dari latar belakang.

Properti CSSDeskripsi Fungsi
max-width: 320px;Membatasi lebar maksimal form, mencegah form memanjang penuh (terlalu lebar) di layar desktop.
padding: 30px;Memberikan ruang napas di dalam kotak agar elemen teks dan input tidak menempel ke tepi.
border-radius: 8px;Menumpulkan keempat sudut tajam pada kotak menjadi melengkung agar UI terlihat lebih modern.
box-shadowMenambahkan efek bayangan (X, Y, blur, warna) pada kotak untuk memberikan kesan kedalaman/mengambang.
 
.login-box {
    width: 100%; max-width: 320px; padding: 30px;
    background: white; border-radius: 8px; box-shadow: 0 4px 10px rgba(0,0,0,0.1);
}
h2 { text-align: center; margin-bottom: 20px; color: #333; }
Step 3 Styling Input & Label

Memberikan ruang (padding) pada input dan mengatur label agar diletakkan dengan rapi di atas kolom isian, bukan di sebelahnya.

Properti CSSDeskripsi Fungsi
display: block; (pada label)Membuat elemen mengambil satu baris penuh, secara otomatis mendorong elemen input ke baris bawah.
margin-bottom: 15px;Memberikan jarak vertikal yang konsisten ke arah bawah antar setiap grup form input.
width: 100%; (pada input)Membuat kolom isian melebar maksimal hingga memenuhi ketersediaan ruang kontainer grup.
border: 1px solid #ddd;Mendefinisikan gaya garis tepi input (ketebalan 1px, model solid, warna abu-abu terang).
display: flex; gap: 8px;Menyelaraskan kotak centang (checkbox) dan teks label secara horizontal dengan jarak yang rapi.
 
.form-group { margin-bottom: 15px; }
.form-group label { display: block; margin-bottom: 5px; font-size: 14px; }
.form-group input {
    width: 100%; padding: 10px; border: 1px solid #ddd; border-radius: 4px;
}
.form-check { display: flex; align-items: center; gap: 8px; margin-bottom: 15px; }
.form-check label { font-size: 14px; margin-bottom: 0; }
Step 4 Final: Tombol & Interaksi

Langkah terakhir adalah memberikan warna pada tombol (Call-to-Action) dan menyematkan efek hover sebagai respon interaktif sistem.

Properti CSSDeskripsi Fungsi
cursor: pointer;Mengubah kursor panah mouse menjadi indikator "tangan menunjuk" (pointer) saat hover di atas tombol.
border: none;Menghapus seluruh garis tepi (border) standar yang biasanya diberikan secara otomatis oleh peramban.
:hover (Pseudo-class)Selektor keadaan khusus yang diaktifkan ketika user meletakkan kursor mouse tepat di atas elemen tersebut.
 
button {
    width: 100%; padding: 10px; background: #2563eb; color: white;
    border: none; border-radius: 4px; cursor: pointer; font-weight: bold;
}
button:hover { background: #1d4ed8; }
